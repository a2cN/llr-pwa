# LLR Terminal PWA — Markdown エディタ強化 調査・実装立案 (v3)

策定日: 2026-07-11 / 前提: モバイルファースト・日本語入力(IME)主体・既存単一 `index.html` 構成

## 0. 現状と課題

現状は素の `<textarea>`。Markdown はプレーンテキストとして書くのみで、
- 記法（見出し・リスト・表・リンク）を手打ちする必要がある
- 書いた結果の見た目を投稿前に確認できない（プレビューがない）
- リスト入力時の改行で `- ` を毎回手打ち

一方で **素の textarea であることが既存機能の土台**になっている点に注意:
- 音声入力（`selectionStart` へ挿入）
- テンプレ/PDF挿入（`insertTextAtCursor`）
- 下書き自動保存（`.value` の読み書き）
- 日本語IME・undo/redo・モバイルキーボードのネイティブ動作

## 1. 調査結果（候補比較）

| 候補 | 方式 | サイズ | モバイル | 日本語IME | 備考 |
|---|---|---|---|---|---|
| **A. 自作ツールバー+プレビュー** | 素のtextarea維持 | +0KB | ◎ ネイティブ | ◎ ネイティブ | marked+DOMPurify は導入済みで流用可 |
| **B. TinyMDE** | contenteditable | ~90KB | ○ モバイル向け設計(CM5の弱点対策が開発動機) | ○ ブラウザ標準のIME処理（実機確認要） | MIT・活発(v0.2.31, 2026-06)・インライン整形表示 |
| **C. OverType** | 透明textarea重ね | ~117KB | ○ | **△ 未検証リスク**（透明文字上のIME変換表示が不明） | MIT・等幅フォント必須・画像インライン表示不可 |
| **D. EasyMDE** | CodeMirror5 | ~280KB | △ モバイル既知問題あり | △ | モバイル非推奨 |
| **E. Toast UI Editor** | 専用エンジン | ~600KB+ | ○ | ○ | 重量級。PWAの軽量方針に不適 |

- D/E は除外（モバイル問題 or 重量）。
- C(OverType) は思想は理想的だが、**「textarea文字を透明化して背面プレビューを重ねる」方式が日本語IMEの変換中表示と相性が悪い可能性**があり、日本語主体の本アプリでは検証コストとリスクが高い。等幅フォント必須も日本語と相性が悪い（全角半角混在で桁ズレ）。
- B(TinyMDE) は「モバイルで文章を書く」ことを目的に作られており、太字は太く・見出しは大きくインライン表示される（真のリッチ化）。ただし contenteditable ベースのため、既存の textarea 連携（音声・テンプレ・PDF・下書き）を全て TinyMDE API (`setContent`/`wrapSelection`/イベント) に付け替える必要があり、日本語IME・Android GBoard の挙動は実機確認が必須。

## 2. 実装方針（2段階）

### Phase 1（推奨・先行実装）: 自作ツールバー + ライブプレビュー
既存 textarea を維持したままリッチ化。**リスクゼロで即効性が高い。**

1. **書式ツールバー**（textarea上部、横スクロール可）
   - B / I / S（太字・斜体・打消し）: 選択範囲を `**..**` 等で囲む。未選択ならマーカー挿入しカーソルを内側へ
   - H2 / H3: 行頭に `## ` をトグル
   - リスト / 番号リスト / チェックボックス: 行頭トグル（複数行選択対応）
   - リンク・コード・引用・表(雛形挿入)・水平線
   - 挿入は `document.execCommand('insertText')` を優先使用（**ネイティブundo履歴が壊れない**。iOS/Android対応）。不可なら `setRangeText` フォールバック
2. **スマートリスト**: `- ` `1. ` `- [ ] ` の行で Enter → 次行に自動継続。空項目で Enter → リスト終了（マーカー削除）
3. **プレビュー切替**: Edit / Preview タブ（またはボタン）。導入済みの marked + DOMPurify + `.wiki-content` スタイルをそのまま流用
4. モバイル配慮: ボタン44px角・`touchend`での確実なフォーカス維持・キーボード上部に収まる1行バー

工数: ~1日 / 依存追加: なし / CSP変更: なし / 既存機能への影響: なし

### Phase 2（任意・別ブランチで実機検証）: TinyMDE 置換トライアル
Phase 1 で不足を感じた場合のみ、`feature/tinymde` ブランチで:
1. TinyMDE + CommandBar を jsdelivr から導入（CSP変更不要圏内）
2. 既存連携の付け替え: 音声入力→`paste()`、テンプレ/PDF→`setContent`、下書き→`getContent()`+changeイベント
3. **実機確認項目**: 日本語IME変換・確定、Android GBoard、undo/redo、カーソルジャンプ、コピペ
4. 問題なければ main へ、問題があれば Phase 1 を継続採用

工数: 1〜2日（検証込み） / 依存: tiny-markdown-editor ~90KB
